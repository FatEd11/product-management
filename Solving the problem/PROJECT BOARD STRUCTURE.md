#PROJECT BOARD STRUCTURE##Overarching roadmap project board. Card/issues represent high-level concepts/projects. They contain:-	The project brief document-	A high level checklist for tasks in this project (potentially associated with issues in specific project board â€“ see below)-	A link to the specific board##Specific project boardCards/issues represent tasks that need to be completed specific to that project. They contain a description of the issue to be overcome, and any associated time limits.Cards can be categorised by colour or similar to distinguish between:1.	Design2.	Build3.	QA4.	Beta5.	Full release6.	Post release
